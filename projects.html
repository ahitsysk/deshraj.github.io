EvalAI: Evaluating state of the art in AI

Leading EvalAI: An open source project which aims to build a centralized platform to host, participate and collaborate in AI challenges organized around the globe and we hope to help in benchmarking progress in AI. Some of the key points which differentiates it from other challenge hosting platforms such as Kaggle, Codalab, TopCoder etc:

  • Support for custom evaluation protocols and phases
  • Parallelized evaluation using map reduce framework
  • Streamlined challenge hosting through UI and zip configuration
  • Centralized leaderboard: Everyone can send their results to EvalAI

Visual Chatbot: A chatbot that can see!

Built a visual chatbot which can hold a meaningful dialog with humans natural, conversational language about visual content. Specifically, given an image, a dialog history, and a question about the image, the chatbot has to ground the question in image, infer context from history, and answer the question accurately.

Visual Question Answering (VQA) Demo

In Visual Question Answering, given an image and a free-form natural language question about the image (e.g., "What kindof store is this?", "How many people are waiting in the queue?", "Is it safe to cross the street?") the machine's task is to automatically produce a concise, accurate, free-form, natural language answer ("bakery", "5", "Yes"). This demo is implemented using Hierarchical Question-Image Co-Attention model.

Gradient-weighted Class Activation Mapping(Grad-CAM) Demo

Gradient-weighted Class Activation Mapping (Grad-CAM) is a novel class-discriminative localization technique, that can be used to make CNN based models interpretable. Grad-CAM highlights regions of the image the VQA model looks at while making predictions. Given an image and a caption or question about that image, the model shows where it looked while doing prediction.

Diverse Beam Search Demo

Beam search, the standard work-horse for decoding outputs from neural sequence models like RNNs produces generic and uninteresting sequences. This is inadequate for AI tasks with inherent ambiguity — for example, there can be multiple correct ways of describing the contents of an image. The demo overcomes this by proposing a diversity-promoting replacement, Diverse Beam Search that produces sequences that are significantly different — with runtime and memory requirements comparable to beam search.
